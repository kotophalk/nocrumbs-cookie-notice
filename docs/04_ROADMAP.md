# Roadmap (Пошаговый план разработки)
**Проект:** NoCrumbs Cookie Notice
**Версия:** MVP

Документ описывает детализированный, пошаговый процесс разработки плагина на основе PRD и SDD. Разработка разбита на 5 спринтов (фаз) от настройки окружения до финального тестирования.

## Фаза 1: Базовая структура и безопасность (Backend)
**Цель:** Создать каркас WordPress плагина, который безопасно инициализируется.
- [x] **Шаг 1.1:** Создать корневую папку `/nocrumbs-cookie-notice/`.
- [x] **Шаг 1.2:** Создать файл `nocrumbs-cookie-notice.php`. Заполнить WordPress Plugin Header комментариями (Название, Версия, Описание).
- [x] **Шаг 1.3:** Добавить защиту от прямого доступа в `nocrumbs-cookie-notice.php` (`if ( ! defined( 'ABSPATH' ) ) exit;`).
- [x] **Шаг 1.4:** Создать пустые файлы `index.php` во всех папках плагина для защиты директорий (directory index.php blocking).
- [x] **Шаг 1.5:** Инициализировать подключение файла класса (через `require_once`) и запуск хука `plugins_loaded`.

## Фаза 2: Бизнес-логика (Backend - PHP OOP)
**Цель:** Реализовать логику вывода в футер только для клиентской части.
- [x] **Шаг 2.1:** Создать файл `includes/class-nocrumbs-cookie-notice.php`. Объявить класс `No_Crumbs`.
- [x] **Шаг 2.2:** Внутри класса создать метод `init()`, регистрирующий хуки `wp_enqueue_scripts` и `wp_footer`.
- [x] **Шаг 2.3:** Написать метод `enqueue_assets()`. Реализовать строгую проверку `if ( is_admin() ) { return; }`, после чего вызывать `wp_enqueue_style` и `wp_enqueue_script`.
- [x] **Шаг 2.4:** Написать метод `render_notice()`. Сделать проверку `is_admin()`.
- [x] **Шаг 2.5:** В методе `render_notice()` получить ссылку на политику через `get_permalink( get_option('wp_page_for_privacy_policy') )`.
- [x] **Шаг 2.6:** Собрать и вывести (через `echo`) HTML-фреймворк плашки со статическим классом `nc-hidden` для скрытия (чтобы избежать моргания до работы JS).

## Фаза 3: Визуальная часть (Frontend CSS)
**Цель:** Сверстать аккуратную минималистичную карточку в углу экрана.
- [x] **Шаг 3.1:** Создать файл `assets/css/nocrumbs-cookie-notice.css`.
- [x] **Шаг 3.2:** Задать стили для `#nc-cookie-banner` (`position: fixed`, `bottom: 20px`, `left: 20px`, `z-index: 9999`).
- [x] **Шаг 3.3:** Добавить "Card Style" (`background-color: #ffffff`, `border-radius: 8px`, `box-shadow`, `padding`). Внутренний сброс стилей типографики (системные шрифты).
- [x] **Шаг 3.4:** Описать класс `.nc-hidden`. Реализовать скрытие через `opacity: 0` / `transform: translateY(20px)` и `pointer-events: none`.
- [x] **Шаг 3.5:** Добавить `transition` для плавного перетекания свойств (анимация появления и исчезновения).
- [x] **Шаг 3.6:** Проверить адаптивность через media queries (на мобилках менять `bottom` и `width`, чтобы плашка не перекрывала весь экран, а аккуратно прижималась вниз).

## Фаза 4: Логика взаимодействия (Frontend JS)
**Цель:** Написать Zero Bloat Vanilla JS коннектор для управления видимостью и cookie.
- [x] **Шаг 4.1:** Создать файл `assets/js/nocrumbs-cookie-notice.js`. 
- [x] **Шаг 4.2:** Добавить EventListener на `DOMContentLoaded` для старта скрипта.
- [x] **Шаг 4.3:** Написать функцию проверки куки функцией `document.cookie.includes('nc_accepted=1')`.
- [x] **Шаг 4.4:** Если куки нет — найти DOM элемент плашки и удалить у него класс `.nc-hidden` (запуская плавное появление).
- [x] **Шаг 4.5:** Найти кнопку "ОК" и повесить `click` listener.
- [x] **Шаг 4.6:** В listener клика записать куки `document.cookie = "nc_accepted=1; max-age=2592000; path=/; SameSite=Lax"` (на 30 дней).
- [x] **Шаг 4.7:** Также в listener клика добавить элементу плашки класс `.nc-hidden` обратно, чтобы она плавно исчезла (можно использовать `setTimeout` для окончательного сокрытия `display: none` после конца transition).

## Фаза 5: Локализация и Завершение
**Цель:** Закончить интернационализацию и подготовить код к релизу.
- [x] **Шаг 5.1:** Проверить, что все строки в PHP вызове HTML-баннера обернуты в `esc_html__()`.
- [x] **Шаг 5.2:** Создать файл `languages/nocrumbs-cookie-notice.pot` для переводчиков.
- [x] **Шаг 5.3:** Сделать финальный аудит размера (JS/CSS).
- [x] **Шаг 5.4:** Провести Manual Testing в рабочем браузере (со сбоями и без).
