# Product Requirements Document (PRD)

**Проект:** No Crumbs — Минималистичный плагин уведомлений о cookie для WordPress (152-ФЗ)
**Версия:** MVP (v1.0.0)

## 1. Product Overview (Краткое описание системы)
No Crumbs — это сверхлегкий плагин для WordPress, предназначенный для закрытия юридических рисков владельцев сайтов в РФ (152-ФЗ). Он решает одну единственную задачу: показывает лаконичное и ненавязчивое уведомление об использовании файлов cookie и собирает согласие пользователя. Главное преимущество плагина перед конкурентами — абсолютная "невесомость" (никакого jQuery, тяжелых стилей или внешних шрифтов), концепция "установил и забыл", а также 100% готовность для российского рынка без избыточного функционала GDPR.

## 2. User Stories (Пользовательские сценарии)

### Владельцы сайтов / малый бизнес
* **Как владелец сайта**, я хочу *установить и активировать плагин без сложных настроек*, чтобы *быстро закрыть вопрос соответствия закону 152-ФЗ и избежать штрафов Роскомнадзора*.

### Веб-разработчики / Студии
* **Как веб-студия**, я хочу *иметь легковесный плагин с чистым Vanilla JS и CSS*, чтобы *обеспечить юридическую безопасность клиентских сайтов без ущерба для оценок Google PageSpeed Insights*.

### Посетители сайта
* **Как посетитель сайта**, я хочу *увидеть аккуратную минималистичную плашку с уведомлением в нижнем углу экрана*, чтобы *нажать "ОК" и плашка больше не мешала мне знакомиться с контентом*.
* **Как посетитель сайта**, я хочу *иметь возможность перейти по ссылке в уведомлении*, чтобы *прочитать "Политику конфиденциальности" и узнать, какие данные собираются*.
* **Как посетитель сайта**, я хочу *чтобы сайт запоминал мой выбор*, чтобы *мне не приходилось кликать на кнопку "ОК" при каждом переходе на новую страницу или при повторном визите завтра*.

## 3. Functional Requirements (Функциональные требования)

### 3.1. Рендеринг баннера (Фронтенд)
* **Откуда берем данные / Текст и Контент (РФ, 152-ФЗ):** 
    * Текст предупреждения захардкожен на русском языке (это важно), должен быть вежливым и коротким. Пример: *"Мы используем файлы с данными (cookie) для работы сайта."*
    * Ссылка на страницу "Политика конфиденциальности" (берется автоматически из базовых настроек WordPress — `wp_page_for_privacy_policy`).
* **Как обрабатываем:** 
    * Сервер (PHP) с помощью хука `wp_footer` добавляет HTML-разметку баннера в конец `<body>`. **Внимание:** обязательна строгая проверка `! is_admin()`, чтобы железобетонно исключить рендеринг плашки внутри панели администратора.
    * Клиентский JavaScript при загрузке страницы проверяет наличие cookie-файла согласия (например, `nc_accepted=1`).
    * Если cookie нет, JS отображает баннер.
    * При клике на кнопку "ОК" скрипт записывает cookie с флагом согласия.
* **Куда выводим / Дизайн (CSS & JS):** 
    * Небольшая, аккуратная "плашка" (card style), изолированный HTML-блок (div) поверх остального контента (`position: fixed`).
    * **Расположение:** Левый нижний угол экрана.
    * **Цвет и стиль:** Минималистичный, ненавязчивый дизайн. Цвет фона — белый (`#ffffff`), с легкой тенью (`box-shadow`) для отделения от основного контента.
    * **Анимация (Vanilla JS):** При клике на кнопку "ОК" плашка плавно исчезает (используется CSS transition).

### 3.2. Настроечный интерфейс (Бэкенд - Admin UI)
* **Откуда берем данные:** В MVP *не требуется отдельная страница настроек*.
* **Как обрабатываем:** 
    * Плагин автоматически ищет стандартную страницу политики конфиденциальности WordPress (`Настройки -> Конфиденциальность`).
    * Если она задана, вставляет ссылку на неё в текст баннера. 
    * Если страница не выбрана в WP, баннер выводится без кликабельной ссылки (или ведет на `#`).

## 4. Non-Functional Requirements (Нефункциональные требования)

* **Скорость и вес (Zero Bloat):** 
    * Общий размер фронтенд-ассетов (JS + CSS) **строго меньше 10 Кб**.
    * JS написан на чистом Vanilla JS (ECMAScript 6+). Использование jQuery запрещено.
    * CSS использует только системные шрифты (`font-family: system-ui, -apple-system, sans-serif;`), без подгрузки сторонних библиотек (Google Fonts, FontAwesome).
* **Срок хранения решения:** Cookie согласия устанавливается сроком на 30 дней.
* **Адаптивность:** Плашка должна корректно отображаться на мобильных устройствах, аккуратно занимая необходимое место внизу экрана, и ни в коем случае не перекрывать весь экран, сохраняя доступ к основному контенту.
* **Изолированность (CSS):** Все классы плагина должны иметь уникальный префикс (например, `nc-cookie-notice-`), чтобы исключить конфликты CSS с активной темой WordPress. Отсутствие "войн z-index".
* **Локализация:** Плагин должен поддерживать стандарты интернационализации WP (i18n, функции `__()`, `esc_html__()`), с базовым русским и английским переводом (pot-файл).

## 5. Out of Scope (Что МЫ НЕ ДЕЛАЕМ в первой версии)

* **GDPR compliance features:** Отложенная загрузка скриптов аналитики (Yandex Metrika, Google Analytics) до получения явного согласия пользователя. В РФ (в рамках простых проектов) часто достаточно факта уведомления.
* **Гранулярные настройки согласия:** Чекбоксы для выбора типов согласия (маркетинговые, технические, аналитические).
* **Сложный UI настроек (Customizer/Admin page):** Настройки цветов баннера через ColorPicker, возможность загружать логотип, выбор позиции через админку. В MVP используем один идеальный, нейтральный, встроенный дизайн.
* **Логирование и сбор статистики кликов:** Мы не ведем учет, сколько раз пользователи нажали на кнопку, и не пишем это в базу данных (избегаем раздувания БД и снижения производительности).
* **Поддержка старых браузеров:** Не поддерживаем Internet Explorer 11 и ниже. Полагаемся на современные стандарты веб-разработки браузеров (Chrome, Safari, Firefox последних лет).
